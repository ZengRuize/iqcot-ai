### 表驱动监督层的 `5 us` 参数提交延迟验证

为检验训练标签是否能进入派生开关级模型，本文在
`four_phase_iek_dynamic_load_refslew.slx` 中实现了表驱动监督层验证。
AI/table 层只选择低维参数 `T_slew`，IQCOT 内环仍负责快速面积事件触发和相位调度。
对于固定斜率基线，参考轨迹视为预提交，在切载时刻开始；对于表驱动策略，
参考轨迹从 `t_load_step+5 us` 开始，以等效模拟 FPGA 推理、同步和参数提交延迟。
在四相 `500 kHz` 条件下，`5 us` 约等于 `10` 个 IQCOT 事件。

该验证覆盖 `40A->20A`、`40A->10A`、`40A->near-0A` 三种切载和五类策略，共
`15` 个开关级工况。结果显示，固定 `40 us` 与固定 `80 us` 的平均 base score
分别为 `9.856` 与 `9.435`；base-score table 采用 `80/80/60 us`，平均 base
score 为 `8.960`；`alpha=0.05` table 采用 `30/50/60 us`，平均 base score 和
`score+0.05T_settle` 分别为 `8.383` 与 `9.657`；`alpha=0.10` table 采用
`30/30/30 us`，在强恢复时间惩罚下取得最低 `score+0.10T_settle=10.785`。

该结果支持一个克制但重要的结论：PIS-IEK 生成的参考斜率标签不仅是离线后处理，
还可以作为延迟提交的表驱动监督层进入派生开关级模型，并继续呈现目标敏感排序。
但这仍不是神经网络 AI-in-the-loop，也不是硬件验证；它只能说明下一步训练 AI
监督层有明确、可运行的开关级验证接口。
